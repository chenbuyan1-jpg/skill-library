# M2 场景脚本集 · 模板

> 输出文件：`PRD/01-scenarios.md`
> 用途：让 AI 知道"用户具体怎么用"，用 30 秒可演示的脚本描述

---


---

## 写作规则

### 场景数量
- **3-5 个**——少了不全面，多了是范围爆炸
- 优先覆盖：核心 happy path + 1-2 个变体场景

### 操作脚本要逐步骤
- ❌ "用户使用 AI 完成任务"——黑盒，AI 会自己脑补
- ✅ 每一步都是可观察的具体动作（点击、输入、看到、复制）

### 失败兜底必须有
- AI 应用 80% 的 bug 来自这里
- 至少列 3 种最常见的失败：网络断 / 输入错 / 服务报错 / 超时 / 限流 / ...

### AI 产品的"理想输出样例"
- **直接给一段示例文本**——让 AI 照着抄结构和风格
- 不是描述"应该长这样"，是真的写出来

---

## 写作范例（仅供参考脚本的颗粒度）

```
## 场景 1：周五出周报

**用户故事**：作为运营，我想把本周聊天群里的精华讨论汇总，以便发给主管。
**前置状态**：用户已选好群、时间范围（最近 7 天）

**操作脚本**：
1. 用户在工具页面贴上 7 天聊天记录（粘贴框，最长 50 万字）
2. 系统显示"分析中..."进度条
3. 30 秒内，输出 3 段：本周 3 件大事 / 5 个待跟进 / 1 个亮点引用
4. 用户点"复制为飞书消息格式"

**期望产出**：3 段总结 + 复制后能粘贴到飞书直接发的格式

**失败兜底**：
- 字数超过 50 万 → 提示分批 + 给"按时间切分"按钮
- LLM 超时 → 显示"分析失败"，但聊天记录保留 + [重试]
- 总结生成空白 → 退化为"按发言人统计 top 5 + 关键词词云"
```
