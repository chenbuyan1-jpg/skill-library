# M5 验收脚本 · 模板

> 输出文件：`PRD/04-acceptance.md`
> 用途：让 AI 写完代码后能机械地判断"是否完成"

---

## 模板

```markdown
# 验收脚本

## 5.1 happy path 验收

### 验收 1: {{名字}}
- **GIVEN**: {{前置状态}}
- **WHEN**: {{用户操作}}
- **THEN**: {{可观察的具体结果，含 DOM/数据/网络/状态}}

### 验收 2: ...

## 5.2 边界 / 错误验收

### 错误场景 1: {{名字}}
- **GIVEN**: {{失败前置，如网络断 / 输入错 / 限流}}
- **WHEN**: {{用户操作}}
- **THEN**: {{兜底行为}}

### 错误场景 2: ...

## 5.3 [AI 产品] 评测脚本

- **评测集位置**: `evals/eval-set.jsonl`
- **样本数**: ≥ {{N}}
- **覆盖类别**:
  - 正向: {{N}} 条
  - 边缘: {{N}} 条
  - 失败: {{N}} 条
  - 对抗: {{N}} 条

- **评测维度**:
  - 维度 1: {{准确性 / 一致性 / 长度合规 / ...}} —— 通过线 ≥ {{X}}
  - 维度 2: ...

- **整体通过线**: 评测集通过率 ≥ {{X%}}

- **bad case 收集机制**:
  - 上线后用户反馈"不好"的样本 → 写入 `evals/bad-cases.jsonl`
  - 每周从 bad-cases 抽 N 条加入正式评测集

## 5.4 完成标志

{{一句具体的、用户可观察的现象}}
```

---

## 写作规则

### GIVEN / WHEN / THEN 三段必写
- 缺一段 = 不可执行
- AI 拿到只有 WHEN 的"用户点击保存"会自己脑补 GIVEN，可能不对

### THEN 必须是「可观察现象」
- ❌ "用户感到满意"
- ❌ "数据正确"
- ❌ "界面美观"
- ✅ "页面右上角显示 toast '保存成功'，2 秒后消失"
- ✅ "数据库 messages 表新增一行，content 字段等于用户输入的文本"
- ✅ "网络面板看到 POST /api/save 返回 200"

### 每个绿色功能 ≥ 1 happy + 1 错误
- 不能只写 happy path
- 错误验收和 happy path 同样重要

### 完成标志只有 1 句
- 这是从完成定义卡 Q2 继承下来的
- 多了等于没有

## 写作范例

```markdown
## 5.1 happy path

### 验收 1: 上传到出纪要
- **GIVEN**: 用户登录，有一份 25MB 的 30 分钟周会录音 mp3
- **WHEN**: 用户点击"上传"，选择该文件
- **THEN**:
  - 上传进度条出现，显示百分比
  - 上传完成后状态变为"转写中"
  - 10 分钟内页面显示四段式纪要 JSON
  - 4 段每段非空，行动项均含 [负责人] tag

## 5.2 错误验收

### 错误场景 1: 超大文件
- **GIVEN**: 用户选了 50MB 文件（超过 30MB 限额）
- **WHEN**: 点击"上传"
- **THEN**:
  - 上传按钮禁用
  - 输入框下方红色提示 "超过 30MB 限制"
  - 不发起任何网络请求

## 5.3 评测脚本

- 评测集 30 条 (见 evals/eval-set.jsonl)
- 覆盖: 正向 15 / 边缘 8 / 失败 4 / 对抗 3
- 维度:
  - 议题召回率 ≥ 90%
  - 行动项含负责人率 = 100%
  - 长度合规（每段 ≤ 300 字） ≥ 95%
- 通过线: 整体通过率 ≥ 85%

## 5.4 完成标志

用户能上传一份会议录音，10 分钟内拿到 4 段式纪要并复制走。
```

## 自查（M5 部分 4 项）

- [ ] 每条 GIVEN / WHEN / THEN 三段齐全
- [ ] THEN 是可观察现象（不是"用户满意"）
- [ ] 完成标志只有 1 句
