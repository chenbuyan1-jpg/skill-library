# M3 范围三色清单 · 模板

> 输出文件：`PRD/02-scope.md`
> 用途：明确告诉 AI"做什么 / 可选 / 绝对不做"，避免范围滑坡

---

## 模板

```markdown
# V1 范围三色清单

## 🟢 绿色（V1 必做）
- [ ] {{功能 1}}
- [ ] {{功能 2}}
- [ ] {{功能 3}}
- [ ] {{...}}

（≤ 8 项）

## 🟡 黄色（V1 可选 / 时间允许就做）
- [ ] {{功能 A}}
- [ ] {{功能 B}}

## 🔴 红色（V1 明确不做，违反视为越界）
- {{功能 X}}：{{原因}}
- {{功能 Y}}：{{原因}}
- {{功能 Z}}：{{原因}}
- {{...}}

（≥ 5 项，每项必有原因）
```

---

## 写作规则

### 红色清单 ≥ 5 项
**这是 PM 纪律，不是 AI 纪律。**

红色清单越多，项目越能锁定。常见红色项的来源：
- 完成定义卡的 Q3"绝对不做"全部转入这里
- 用户调研里"想做但暂时不做"的功能
- 你自己想做但理性告诉你"不该现在做"的功能
- AI 容易自动加戏的扩展（如"添加分享功能""加上数据看板"）

### 红色每项必有原因
- ❌ "多人协作：暂时不做"
- ✅ "多人协作：当前自用，引入会让数据模型复杂 3 倍"
- ✅ "导出 PDF：现有需求是给人看的纪要，没有归档需求；如果 V2 客户要求再加"

### 绿色 ≤ 8 项
- 超过 8 项说明你在堆砌，没在做 MVP
- 绿色 = 必做，必做的事多了就做不完
- 拿不准的功能放黄色，不要塞绿色

### 三色之间不能重复
- 一个功能只能在一种颜色里
- 不能"既是必做又是可选"

## 写作范例

```
## 🟢 V1 必做
- [ ] 上传 mp3/m4a 录音（≤ 30MB）
- [ ] 调用第三方语音转文字 API
- [ ] 调用 LLM 生成四段式纪要
- [ ] 展示并允许编辑纪要
- [ ] 复制 Markdown / 导出 .md

## 🟡 V1 可选
- [ ] 历史纪要列表
- [ ] 自定义 prompt 模板

## 🔴 V1 不做
- 实时字幕：技术栈完全不同（流式语音 vs 批处理）
- 多人协作：数据模型复杂度会 ×3，且当前用户都是单兵作战
- 移动端适配：用户主要在工位
- 语音说话人识别：API 成本翻倍且效果不稳
- 集成钉钉/飞书：需要走平台审核，时间不可控
```
