# PRD 终审 · 22 项自查清单

> 写完 PRD 后逐项检查。**全 yes 才算合格**，一项 no = 回去改。

---

## M1 项目身份卡（3 项）

- [ ] 一句话定义真的是一句话（无"和"/"以及"/"等"）
- [ ] 目标用户写到 persona 级（能脑补出具体的人）
- [ ] "不解决" ≥ 2 项

---

## M2 场景脚本集（4 项）

- [ ] 场景数量 3-5 个
- [ ] 每个场景逐步骤写（不是黑盒一句话）
- [ ] 每个场景"失败兜底" ≥ 3 种
- [ ] 若 AI 产品：每个场景给"理想输出样例"

---

## M3 范围三色清单（3 项）

- [ ] 红色 ≥ 5 项，每项有原因
- [ ] 绿色 ≤ 8 项
- [ ] 三色之间无重叠 / 矛盾

---

## M4 数据与契约（4 项）

- [ ] 字段表有：类型 / 必填 / 默认值
- [ ] 接口契约写了错误码（不只 happy path）
- [ ] 若 AI 产品：prompt 是完整可复制版本（不是描述）
- [ ] 若 AI 产品：温度 / token 预算 / 失败行为齐全

---

## M5 验收脚本（4 项）

- [ ] 每个绿色功能 ≥ 1 happy path + 1 错误路径
- [ ] 每条 GIVEN / WHEN / THEN 三段齐全
- [ ] THEN 是可观察现象（不是"用户满意"）
- [ ] 完成标志只有 1 句

---

## 跨模块（4 项）

- [ ] 5 个模块在独立文件里（解耦原则）
- [ ] index.md 是真的索引（≤ 1 页）
- [ ] 没有同时混入"该做什么"和"具体怎么实现"
- [ ] 任意一个模块单独丢给 AI，AI 都能产生对应输出

---

## 总分

**全 22 项 yes** = PRD 合格，可以丢给 AI 落地
**任一项 no** = 回去改对应模块
