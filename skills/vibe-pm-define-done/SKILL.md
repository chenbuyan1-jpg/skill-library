# vibe-pm-define-done — 完成定义器

> 所有混乱都源于"完成"定义不清。
> 这张卡是所有后续决策的锚点。任何时候有人说"那这个要不要做"，回头看这张卡就有答案。

---

## 触发条件

**触发**：
- 用户处于 idea 阶段（还没写 PRD、没动代码）
- 关键词："我有个想法 / 想做一个 / 立项 / 开始一个项目 / 让 AI 帮我做 / 完成的定义 / 怎么算做完"

**不触发**：
- 已经有完成定义卡 → 转 `vibe-pm-prd`
- 已经写了 PRD → 转 `vibe-pm-doc-audit`（规划中）
- 已经在改代码 → 转 `vibe-pm-engineering-review`（规划中）
- 单纯讨论/学习/询问，不是项目启动

---

## 核心原则

1. **可观察 > KPI** —— "我能在浏览器里和 3 个角色完成一次反思讨论" 比 "用户留存提升 20%" 有用
2. **排除 > 包含** —— 主动声明"不做什么"比堆砌"做什么"更能锁项目
3. **画面 > 文字** —— 用户能脑补出来的画面，才是真的"完成"

---

## 4 阶段交互协议（严格执行）

```
阶段 0: 触发判定
   ↓
阶段 1: 现状把脉（1 问）
   ↓ 用户回答
阶段 2: 5 道追问（一次只问 1 道）
   Q1 → Q2 → Q3 → Q4 → Q5
   ↓ 全部答完
阶段 3: 输出完成定义卡 + 自动推断项目档位 + 推荐下一步 skill
   ↓ 用户拍板
阶段 4: 锁定 + 持久化（写入 docs/done.md）
```

**铁律**：
- 阶段 2 一次只问一道。一次塞 5 个问题给用户，他会用一句话糊弄过去
- 任何阶段未拍板前，禁止进入下一阶段

---

## 阶段 1：现状把脉

只问一个开放问题：

> "你心里已经清楚的是什么？最不确定的是什么？"

听完用户回答后，进入阶段 2。

---

## 阶段 2：5 道核心追问（按顺序 + 逐条提问）

### Q1：用一句话告诉我你想做什么？

**追问理由**：一句话讲不清，说明项目还没成型。

**不合格答案的特征**：
- 出现"和"/"以及"/"等"——其实是两个项目
- 出现"高效便捷智能"等形容词堆砌——不是定义
- 长度 > 30 字——还没浓缩到本质

**追问示范**：
> "我想做一个 AI 工具帮助产品经理高效写 PRD 并能自动竞品分析" → 这是两个项目（写 PRD + 竞品分析）。这次先做哪一个？

---

### Q2：你脑子里 done 那一刻的画面是什么？请描述一个 30 秒能演示的场景。

**这是 5 题里最重要的**。能脑补画面 = 锁住了完成；画不出 = 项目其实没想清楚。

**不合格答案**：
- "用户能愉快使用" —— 不是画面
- "完成所有核心功能" —— 同义反复
- 描述里没有具体动作 / 屏幕 / 输出

**追问示范**：
> "用户能完成 PRD 撰写" → 画面是什么？打开什么、点什么、看到什么输出？

**3 行风格范例**（仅参考"画面应该多具体"，不要照抄结构）：
- 内部工具：运营在后台粘贴 100 个订单号，点退款，2 分钟后看到 100 条退款成功记录
- 生活工具：我在手机上打开 app，给冰箱里 5 张照片，30 秒后看到一份周末菜单 + 购物清单
- AI 协作：我把会议录音拖进网页，5 分钟后看到 4 段式纪要，复制粘贴到群里

---

### Q3：列出 3 件这次绝对不做的事。

**追问理由**：主动排除是最强的锁。"做什么"会无限滑坡，"不做什么"是真纪律。

**不合格答案**：
- "暂时不做" —— 没真排除
- 不能给出原因 —— 没想清楚
- 列了大家显然不会做的事（如"不做太空旅行"）—— 避重就轻

**追问示范**：
> "暂时不做多人协作" → "暂时"什么时候解除？为什么这次不做？

---

### Q4：你期望多久后能用上它？

**强制四选一**（不允许"尽快""越快越好"等模糊回答）：
- A. 几小时内（同一天）
- B. 几天内（一周）
- C. 几周内（一个月）
- D. 几个月（更长）

---

### Q5：这是你自用，还是给别人用？

**强制四选一**：
- A. 只给我自己用（本机）
- B. 给我和身边几个人（≤10）
- C. 给团队 / 公司（10-1000）
- D. 公开 / 商用（>1000 或对外）

---

## 阶段 3：输出完成定义卡

按以下模板输出（可用 references/card-template.md 作为模板）：

```markdown
# 完成定义卡

> 任何后续讨论中"要不要做 X"的问题，请先回头看本卡。

| 字段 | 内容 |
|---|---|
| 项目一句话 | {{Q1 答案}} |
| 完成画面（30 秒可演示） | {{Q2 答案}} |
| 绝对不做 | 1. {{...}}<br>2. {{...}}<br>3. {{...}} |
| 时间感 | {{A/B/C/D}}: {{时间}} |
| 用户范围 | {{A/B/C/D}}: {{描述}} |
| 项目档位 | {{T0/T1/T2/T3/T4}} |
| 建议下一步 skill | {{...}} |
| 锁定时间 | {{YYYY-MM-DD HH:mm}} |
```

### 项目档位推断规则（Q4 时间 × Q5 范围）

| Q4 \ Q5 | A 自用 | B 几个人 | C 团队 | D 公开 |
|---|---|---|---|---|
| A 几小时 | T0 极速原型 | T0 极速原型 | ⚠️ 不匹配 | ⚠️ 不匹配 |
| B 几天 | T1 个人工具 | T1 个人工具 | T2 小团队 | ⚠️ 不匹配 |
| C 几周 | T1 个人工具 | T2 小团队 | T2 小团队 | T3 中型 |
| D 几个月 | T2 小团队 | T2 小团队 | T3 中型 | T4 大型 |

⚠️ **不匹配区**：必须回 Q4/Q5 重新对齐（不能强行推进）。

### 各档位推荐 skill 路径

| 档位 | 下一步 skill | 跳过的环节 |
|---|---|---|
| T0 极速原型 | 直接写代码（可不做 PRD） | 跳过 PRD / 文档审查 / 评测 |
| T1 个人工具 | `vibe-pm-prd` 简版（M1+M5 必写） | 跳过文档审查 / 灰度 / 监控 |
| T2 小团队 | `vibe-pm-prd` 完整版 | 跳过 BRD / MRD |
| T3 中型 | `vibe-pm-prd` + `vibe-pm-doc-audit` | 跳过深度合规 |
| T4 大型 | 全套 skill 链 | 不跳过 |

---

## 阶段 4：锁定 + 持久化

确认通过后：
1. 写到项目根目录的 `docs/done.md`
2. 在 `docs/done.md` 顶部加固定文案：

   > 本卡为项目锚点。任何后续决策与本卡冲突时，要么修订本卡（明确记录），要么放弃该决策。

3. **不允许**没有正式 review 就修改本卡

---

## 8 项自查（输出前必过）

- [ ] Q1 一句话真的是一句话（无"和"/"以及"/"等"）
- [ ] Q2 画面能让人脑补出具体动作和输出
- [ ] Q3 至少 3 项不做的事，每项有原因
- [ ] Q4 是 A/B/C/D 之一，不是模糊形容
- [ ] Q5 是 A/B/C/D 之一，明确用户数
- [ ] 档位 = Q4 + Q5 的交叉点
- [ ] 时间 vs 范围无冲突（不在 ⚠️ 区）
- [ ] 卡片所有字段都填了，无 TBD / 待定

> 8 项全 yes 才能锁定。

---

## 反模式（同事最常踩的坑）

| 反模式 | 表现 | 修法 |
|---|---|---|
| "以后再定" | "时间感先选 C 吧，到时候看" | 不允许。强制现在选定 |
| 画 KPI 不画画面 | "用户留存 ≥ 30%" 当作完成标志 | 拒绝。让用户描画一个具体的 demo |
| "暂时不做" | "多人协作暂时不做" | 拒绝。用"V1 不做"代替 |
| 野心匹配不上时间 | 选 A 几小时 + D 公开 | 强制重新对齐 |
| "全都要" | Q1 答案塞 3 个项目 | 砍成 1 个，剩下进 V2/V3 |
| 跳过 Q3 | "不做的事让我想想再说" | 拒绝推进。Q3 没答 = 没锁定 |

## 与其他 skill 的衔接

```
vibe-pm-define-done
        ↓ 输出 docs/done.md
        ↓
   ┌────┴─────────────────┐
   ↓                       ↓
T0 直接写代码        T1+ 进入 vibe-pm-prd
                          ↓
                  PRD 中的 M1 自动继承本卡内容
                  PRD 中的 M3 红色清单 ⊇ 本卡 Q3
                  PRD 中的 M5 完成标志 = 本卡 Q2
```

→ 本 skill 是 PRD 的**输入**。PRD 不重复问 define-done 已问过的问题。

---

## 给同事的 1 分钟上手

1. **任何 vibe coding 项目，第一件事就是这个 skill** —— 不是 PRD、不是设计、不是想技术栈
2. **5 个问题不是表单，是对话** —— 卡在哪题就在哪题深入
3. **卡片锁定后钉在墙上**（虚拟意义） —— 任何决策都要回头看一眼
