# 面试前作战准备 Skill

本 skill 用于面试前：把 JD、公司现状、岗位真实问题和候选人自身经历接起来，形成可执行的面试打法。

与 `interview-reviewer` 的边界：

- `interview-prep-strategist`：面试前，根据公司/JD/候选人画像准备打法。
- `interview-reviewer`：面试后，根据转录文本复盘失分、训练回答。

## 必读资料

执行前先读取：

1. `../interview-reviewer/references/user-profile.md`：候选人画像、薪资背景、风险偏好。
2. `../interview-reviewer/references/project-registry.md`：候选人项目证据库。
3. `../interview-reviewer/references/salary-level-ladder.md`：20万/30万/35万/40万/50万+ 回答档位。
4. `../interview-reviewer/references/knowledge-vault-index.md`：候选人 Obsidian 知识库索引。
5. `references/prep-output-framework.md`：固定输出骨架。

按需读取：

- `references/company-research-framework.md`：公司调研口径。
- `references/jd-salary-level-mapping.md`：JD 薪资和岗位责任档位映射。
- `references/interview-question-prediction.md`：预测面试题方法。
- `references/candidate-fit-risk-framework.md`：候选人匹配和岗位风险判断。

## 核心原则

1. **面试准备不是公司百科**：所有公司信息都要翻译成"岗位为什么招人、面试官会问什么、候选人怎么答"。
2. **公司现状必须联网验证**：公司官网、招聘页、新闻、融资、产品、社媒口碑、竞品信息都可能变化。涉及当前公司信息时必须搜索验证，并区分"已确认事实 / 合理推断 / 待验证"。
3. **JD 薪资动态映射**：不固定用 30-35 万或 35-40 万。先解析 JD 薪资、城市、年包、职责，再判断岗位明面档位、实际责任档位、候选人支撑档位和表达目标档位。
4. **候选人经历优先**：所有回答策略必须绑定候选人真实项目、角色边界、数据口径和可迁移能力。不能给换个人也能用的模板。
5. **岗位值不值得去必须判断**：结合候选人的成长诉求、产品经验短板、薪资期待、是否有人带、是否有数据/权限/资源，输出"值得去 / 谨慎去 / 不建议去"。
6. **两种产物都要输出**：完整备战文档用于沉淀；面试前速记卡用于临场 30 分钟复习。
7. **不编内部信息**：公司内部组织、真实数据、面试官意图只能基于 JD、公开资料、用户输入或合理推断。推断必须标注。
8. **回答不是背稿**：给结构、立场、证据和追问防御链，让候选人能用自己的话讲。

## 输入收集

如果用户没有一次性给全，按下面顺序用纯文本追问，不用弹窗：

1. 公司名和岗位名。
2. JD 文本、截图、链接或招聘页。
3. 薪资范围和城市；如果 JD 已包含则自动提取。
4. 面试轮次和面试官角色：HR / 业务负责人 / 产品负责人 / 技术算法 / 高管 / 不确定。
5. 用户想重点准备的问题：项目深挖、AI 产品、业务增长、离职原因、谈薪、岗位是否值得去等。
6. 是否需要发布到飞书；没有则保存本地 Markdown 到 `/Users/chenbuyan/Documents/面试`。

## 工作流

```text
Phase 0: 读取候选人画像和项目证据库
  user-profile + project-registry + salary-level-ladder + knowledge-vault-index
    ↓
Phase 1: JD 和薪资解析
  岗位职责 / 硬要求 / 隐性要求 / 薪资范围 / 城市 / 年限 / 组织暗号
    ↓
Phase 2: 公司现状调研
  官网 / 招聘 / 新闻 / 产品 / 融资 / 竞品 / 社媒口碑
  输出: 已确认事实 / 合理推断 / 待验证
    ↓
Phase 3: 岗位真实问题推断
  公司现状 → 岗位为什么出现 → 团队可能缺口 → 面试官可能考什么
    ↓
Phase 4: 候选人匹配与风险判断
  强匹配点 / 可迁移能力 / 短板 / 不能硬碰的问题 / 岗位值不值得去
    ↓
Phase 5: 预测面试题与回答策略
  HR / 业务 / 产品 / 技术 / 高管问题
  每题输出: 考察点 + 不要怎么答 + 核心立场 + 档位回答 + 追问防御
    ↓
Phase 6: 方案准备
  30/60/90 天计划 + MVP 思路 + 数据/资源诉求 + 第一阶段验证方式
    ↓
Phase 7: 组装输出
  完整备战文档 + 面试前速记卡 + 待确认事项 + 审核状态
```

## 输出要求

固定输出结构见 `references/prep-output-framework.md`，不得删除主要目录。没有信息时写"暂无/待确认"。

关键模块必须包含：

- 公司现状与岗位真实问题。
- JD 薪资和岗位责任档位。
- 候选人匹配与短板。
- 预测面试题。
- 按薪资档位的回答策略。
- 针对公司问题的 30/60/90 天方案。
- 反问清单。
- 岗位是否值得去。
- 面试前速记卡。

## 质量审核

发布前必须做三重审核：

1. 公司信息真实性：哪些来自公开资料，哪些是推断。
2. 候选人经历真实性：是否符合候选人真实项目和数据口径。
3. 面试可用性：候选人能不能自然说出口，能不能防住追问。

审核不通过的内容要修正；无法确认的内容放入"待确认事项"。
