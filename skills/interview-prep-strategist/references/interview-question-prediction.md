# 面试题预测框架

目标：根据公司现状、JD、面试官角色和候选人简历，预测最可能出现的问题，并给出回答策略。

## 预测路径

```text
公司现状 → 岗位真实问题 → 面试官角色 → 候选人经历 → 预测问题
```

## 按面试官角色预测

| 面试官 | 重点 | 常见问题 |
|---|---|---|
| HR | 动机、稳定性、薪资、城市 | 为什么离职、为什么看我们、薪资期望、是否接受城市 |
| 业务负责人 | 业务结果、落地能力、行业迁移 | 你来了怎么做、怎么提升指标、过去项目怎么迁移 |
| 产品负责人 | 产品方法、需求判断、优先级 | 为什么先做这个、怎么定义 MVP、怎么评估效果 |
| 技术/算法 | AI 理解、可行性、协作边界 | RAG/Agent/Prompt 怎么落地、怎么评测、怎么和研发协作 |
| 高管 | 战略判断、商业化、组织能力 | 为什么这个方向值得做、怎么规模化、长期壁垒是什么 |

## 公司现状触发题

| 公司/岗位信号 | 面试官可能问 | 候选人准备方向 |
|---|---|---|
| 没有产品团队 | 没人带你怎么推进 | 讲自驱 + review诉求 + 资源边界 |
| AI 工具刚起步 | 第一版怎么做 | 讲业务目标、MVP、数据、评测 |
| 内容电商/增长 | 怎么做爆款内容 | 讲人群、卖点、脚本、投放回流 |
| 外部工具很多 | 为什么不直接采购 | 讲外部能力 + 自有业务资产中间层 |
| 多团队协作 | 怎么推动研发/运营/算法 | 讲目标对齐、需求优先级、验收标准 |
| 薪资高但职责宽 | 你能不能 owner | 讲边界、资源、指标、阶段计划 |

## 预测优先级

优先准备 8-12 道题，不要堆 30 道题。

排序规则：

1. JD 核心职责相关。
2. 公司当前问题相关。
3. 候选人简历最容易被追问。
4. 薪资/定级影响大。
5. 用户特别担心的问题。
