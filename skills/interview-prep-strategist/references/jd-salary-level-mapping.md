# JD 薪资与岗位档位映射

目标：根据 JD 薪资、职责、城市、年限和候选人情况，判断面试表达应该达到哪个档位。

本文件与 `../interview-reviewer/references/salary-level-ladder.md` 配合使用。

## 解析顺序

1. 提取 JD 薪资：月薪、年薪、薪资月数、底薪/绩效/期权。
2. 判断明面薪资档位：按 JD 给出的年包范围。
3. 判断实际责任档位：看职责是否超过薪资写法。
4. 判断候选人经历支撑档位：结合 user-profile 和 project-registry。
5. 判断表达目标档位：面试中必须讲出哪一级能力信号。
6. 判断风险补偿：薪资/title/权限是否覆盖岗位风险。

## 年包换算

```text
月薪 * 薪资月数 = 年包
如果只写 20-35K，默认按 12-13 薪估算，并标注"推断"。
如果有绩效/提成/期权，单独列为不确定部分。
```

## 责任档位暗号

| JD 表述 | 可能责任档位 | 需要追问 |
|---|---|---|
| 独立负责某模块 | 30万级 | 模块边界和指标是什么 |
| 从0到1搭建体系 | 35-40万级 | 是否有资源、数据、决策权 |
| 负责 AI 产品落地和业务结果 | 35-40万级 | 结果指标是谁背 |
| 统筹产品/研发/运营/算法 | 40万级 | 是否有正式 owner 权限 |
| 负责战略/商业化/团队管理 | 50万+ | title、汇报线、预算和团队配置 |
| 什么都要做但薪资普通 | 高风险 | 是成长机会还是填坑 |

## 输出格式

```text
JD 明面薪资档位：
岗位实际责任档位：
候选人当前经历支撑档位：
面试表达目标档位：
最影响涨薪的能力信号：
薪资是否覆盖岗位风险：
```

## 判断原则

- 薪资高不等于值得去；要看是否有成长、权限、数据和资源。
- 薪资低但能补关键 AI 产品履历，也可以继续聊，但不能接受负责人责任和执行岗薪资。
- 如果候选人产品经验不够成熟，必须重点判断是否有人 review、是否有学习对象、是否给试错时间。
- 如果 JD 要候选人主导 AI 体系，回答至少要呈现 35-40 万级：业务目标、MVP、指标、评测、资源协同、迭代闭环。
