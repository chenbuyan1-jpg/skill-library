# 薪资档位回答梯度

用途：把面试回答从"好/不好"进一步判断为"当前像哪个年薪档位"，并给出目标档位的补强方向。薪资档位是表达质量和岗位责任的训练标尺，不是承诺市场报价。

默认按中国互联网/AI产品岗位年包口径分层；如JD薪资、城市、行业明显不同，先说明已按该岗位调整。

## 档位标准

| 年薪档位 | 面试官期待的能力信号 | 回答特征 | 常见扣分点 |
|---------|------------------|---------|-----------|
| 20万级 | 能执行，有基础产品/运营理解 | 能讲清做过什么、负责什么、结果是什么 | 流水账、缺数据、只讲工具使用 |
| 30万级 | 能独立负责模块，有业务结果 | 能讲清问题定义、方案选择、协作推进、上线结果 | 有结果但缺归因，讲不清取舍 |
| 35万级 | 能从业务目标倒推产品方案 | 能把用户、场景、数据、转化链路串起来，能防住2-3轮追问 | 只讲单点功能，不能迁移到新岗位 |
| 40万级 | 能负责复杂项目/小方向，有方法论 | 能讲清指标体系、trade-off、评测机制、资源协调和复盘迭代 | 没有体系感，不能说明为什么先做A不做B |
| 50万+ | 能做负责人视角的业务判断 | 能连接战略、商业化、组织协作、长期壁垒，能提出原创判断 | 只有执行经验，没有业务owner视角 |

## 单题输出格式

不要机械生成5段长话术。采用以下格式，控制信息密度：

```plaintext
【当前回答档位】约30万级：能讲清项目结果，但缺少指标口径和迁移论证。
【目标档位】35-40万级：需要补"为什么这样做/为什么不是外部工具/如何持续评测"。
【档位差异】
- 20万级会说：我做了一个AI文案工具，提高了产能。
- 30万级会说：我负责从需求到上线，让编导从15条提升到40条。
- 35万级会说：我先定位投放素材供需不匹配，再用垂类卖点库和评测机制保证转化不掉。
- 40万级会说：我把爆款内容经验产品化，建立样本库、标签体系、质量评测和bad case闭环。
- 50万+会说：这个项目本质是把内容组织能力变成可规模化的AI生产系统，影响的是人效、投放ROI和组织知识沉淀。
【本题冲刺版回答】给出适合目标薪资档位的一段自然口语回答。
```

## 整场总评格式

```plaintext
当前表达档位：约{20/30/35/40/50万+}年薪级
目标岗位档位：约{从JD或用户期望推断}
核心差距：{一句话指出最影响薪资档位的缺口}
```

## 高频题优先级

至少选择3个最影响薪资的题目输出档位梯度，优先级：

1. 自我介绍
2. 核心项目深挖
3. 离职原因
4. 职业规划/为什么做AI产品
5. 为什么选择这家公司
6. 薪资期望

每题输出：

```plaintext
题目：{面试官问题}
你当前表现：{一句话}
当前档位：{20/30/35/40/50万+}
目标档位回答应该多出什么：{2-3个能力信号}
冲刺版回答：{100-200字，可直接说出口}
```

## 谈薪风险判断

| 情况 | 判断方式 |
|------|---------|
| 当前回答档位 < 当前薪资 | 说明面试表达低估了真实能力，优先补项目证据和结构 |
| 当前回答档位 = 当前薪资 | 说明匹配稳定，但跳薪需要更强owner信号 |
| 当前回答档位 > 当前薪资 | 说明有跳薪空间，建议强化岗位匹配和谈薪底气 |
| 期望薪资高于回答档位 | 明确指出差距，不给虚假鼓励，给出补强任务 |

## 岗位风险判断

如果面试中暴露出组织/岗位风险（如没有产品团队、没人带、目标不清、只招一个人填坑、数据权限不明、薪资低于责任），输出岗位风险判断。

| 维度 | 好信号 | 风险信号 | 需要追问 |
|------|--------|----------|----------|
| 成长 | 有成熟产品负责人review，有复盘机制 | 无人带，只让候选人自己摸索 | 入职后方案由谁review？ |
| 权限 | 能看数据、定优先级、推动研发/运营 | 只写PRD/接需求/背结果 | 产品是否有版本优先级决策权？ |
| 目标 | 3个月目标聚焦且可衡量 | 什么都要做，成功标准模糊 | 试用期最核心指标是什么？ |
| 资源 | 有研发/算法/数据/运营支持 | 没资源但要求结果 | 可调用哪些团队资源？ |
| 薪资 | 薪资匹配责任和风险 | 负责人责任、执行岗薪资 | 薪资是否对应owner职责？ |

输出结论分三档：

- **值得去**：能补关键履历，且有权限/数据/资源/导师或review机制。
- **谨慎去**：能补履历，但需要用薪资或title覆盖风险。
- **不建议去**：没人带、目标不清、无权限无数据，还要求候选人背核心结果。
