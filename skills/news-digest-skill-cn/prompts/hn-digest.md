### 筛选标准

从 100 条新闻中，优先筛选以下类别：

1. **AI 行业动态** — 与 AI 求职市场、开源生态、技术趋势直接相关的新闻
2. **高影响力论文** — 未来可能产生重大影响（high impact）的研究工作
3. **金融市场** — 美股相关的重要变动和分析
4. **AI 实践方法** — 关于如何更好地使用 AI 的经验和方法论
5. **思维模型** — 值得学习的思考方式和认知框架

### 执行 Pipeline（强制使用此流程以保证深度和质量）

为了避免随着阅读的文档增多导致上下文过长、对每条新闻分析不够详细的问题，必须严格按照以下多步 Pipeline 执行任务：

1. **获取批量数据**：使用内置 HN 工具（`uv run {baseDir}/deps/hn/scripts/hn.py top -n 100`）获取 Hacker News 前 100 条热门新闻，并将这些原生数据列表写入到一个临时文件（如 `tmp/hn_top_100.md`）。
2. **筛选高价值新闻**：基于该临时文件，根据下方"筛选标准"选出 20 条与我最相关的候选新闻。
3. **获取详细内容与高赞评论**：编写并运行 Python 脚本，通过 Hacker News API（`https://hacker-news.firebaseio.com/v0/item/{id}.json`）一次性获取这 20 条新闻的具体内容和前几个高赞评论，并为每一条新闻在 `tmp/hn_tasks/` 目录下生成独立的 `.md` 任务文件。
4. **独立并发分析（Sub-agents）**：针对这 20 个任务文件，**必须**使用 `generalist` sub-agent 独立处理。每个子任务只接受单条新闻的内容和当前的"解读要求"，并在干净的上下文中生成一份独立的深度解读报告（如 `*_out.md`）。这能保证 AI 在解读每条新闻时不会因为上下文污染而简略。
5. **合并生成最终报告**：编写脚本读取这 20 份独立分析报告，拼接成一份完整的 Markdown 文件，并写入到下方要求的最终输出路径中。

### 解读要求

对筛选出的每条新闻，必须获取其 Hacker News 原帖的高赞评论，并按以下框架进行深度解读：

1. **核心事实** — 用第一性原理拆解：这件事的本质是什么？这部分内容要详细，确保我看完直接能抓住本质，不会缺少必要的信息
2. **深度分析** — 为什么重要？底层逻辑是什么？
3. **社区高赞讨论** — 总结 Hacker News 原帖中高分/高赞评论的核心观点：大家在讨论什么？存在哪些争议？行内人是如何看待这个问题的？
4. **未来影响** — 这件事会如何发展？对行业/市场有什么潜在影响？
5. **批判性反思** — 是否存在过度炒作（overhype）的嫌疑？信息源是否在刻意渲染某种氛围？结论是否经得起推敲？
6. **一句话本质** — 在所有的分析之后，用一句话极其直白地概括它的底层技术/商业现实。强制使用句式："说穿了，这东西就是个……"。注意：必须保持客观、中立、深刻，旨在洞察事物的核心机制或真正价值，严禁使用一味贬低、阴阳怪气或将其描述为"垃圾/套壳"的戏谑口吻。
7. **对我的启发** — 对我的求职、开源项目选择、投资决策有什么具体启示？

### 输出要求

- 解读质量要足够高，让我读完后不需要再看原文和原评论
- 重结论和洞察，轻信息罗列
- 对有争议的内容要明确标注不确定性，不要伪装成确定结论
- 新闻必须要在合适的地方给出超链接，比如标题下面的第一行，但不可以给标题上，h2h3h4任何级别的都不行！这会导致标题被引用的时候特别长 important！
- 每条新闻标题下方必须同时给出两个链接：第一行是原文链接（URL），第二行是 HN 讨论页链接，格式为 `HN讨论: https://news.ycombinator.com/item?id={story_id}`。story_id 从 HN API 获取的 item id 字段获得。两个链接都不能放在标题上！
- 写入文件路径为 `新闻信息/news/{当前日期} HackerNews.md`（如 `2026-03-04 HackerNews.md`），路径可按需修改
- 关键的缩写、术语要给出解释
